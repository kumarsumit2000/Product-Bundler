# Phase 3 — Discount Function (Rust) with Two-Node Combinability

**Date:** 2026-05-06
**Status:** Approved (pending user spec review)
**Depends on:** Phase 2 ([Bundle + QB CRUD](./2026-05-06-phase-2-bundle-qb-crud-design.md)), [Group A amendments](./2026-05-04-spec-amendments-group-a.md)

## 1. Scope & Goal

Ship the Shopify Function that powers all Bundle and QB discounts at checkout. Per-rule combinability honored properly via **two discount nodes per shop** — one for combinable rules, one for non-combinable. Both nodes point at the same compiled Rust Wasm Function; the Function disambiguates via a `node_kind` metafield attached to each node.

This phase implements **Group B Amendment 1** (proper per-rule combinability) on top of CLAUDE.md §6's MVP design. The simplification described in §6 ("strategy=ALL iff every rule is combinable, else FIRST") is **superseded** by this two-node architecture.

### In scope

- New extension `extensions/discount-function/` (Rust + Cargo + `shopify_function` crate, target `wasm32-wasip1`).
- Function logic split into 4 modules: `config.rs` (deserialization), `matcher.rs` (cart-line matching), `discount.rs` (discount math), `main.rs` (orchestration).
- Function input query (`input.graphql`) targets `cart.lines.discounts.generate.run` (newer line-level API).
- Schema migration `0002`: add `shopifyDiscountIdCombinable` and `shopifyDiscountIdNonCombinable` columns to `shops`. Existing `shopifyDiscountId` retained but unused (cleanup deferred to Phase 8).
- New helper `apps/admin/app/lib/discount-nodes.ts` with `ensureDiscountNodes(admin, db, shopId)` — idempotent, creates missing nodes via `discountAutomaticAppCreate`, persists IDs to D1.
- All 4 bundle/QB save routes (Phase 2) call `ensureDiscountNodes` before `syncShopConfig`.
- Local testing via Rust unit tests + 4-5 fixture inputs run through `shopify app function run`.
- 6 new TypeScript tests for `discount-nodes.ts`. Total at end of Phase 3: 74.
- End-to-end manual smoke test on dev store: create bundle/QB, add to cart, verify discount at checkout.

### Out of scope (deferred)

| Item | Defer reason |
|---|---|
| Metafield sharding (Group B item 2) | Phase 2 has 50KB hard fail in `syncShopConfig`. Add sharding in Phase 8 if production hits it. |
| Pre-indexing by productId (Group B item 3) | Function instruction budget concern is theoretical at v1 scale (~hundreds of rules). Add when bites. |
| Inventory awareness in Function (CLAUDE.md §17.12) | Phase 4 widget hides OOS bundles client-side. Function-side inventory check requires extra cart line query. |
| Cleanup of unused `shops.shopifyDiscountId` column | D1 column drop is risky; defer to Phase 8 polish. |
| Migration for orphaned discount nodes (merchant deletes them via Shopify admin) | Off the happy path; defer until first complaint. |
| CI integration of `shopify app function run` fixtures | Run locally during dev. CI integration in Phase 8. |

### Why two-node architecture (recap from brainstorm)

Per-bundle/per-QB `combinable: boolean` cannot be honored within a single discount node — a node has one `combinesWith` setting that applies to all discounts it returns. With two nodes:
- **Node "combinable":** `combinesWith.{product,order,shipping}Discounts = true`. Tagged with metafield `node_kind: "combinable"`.
- **Node "non_combinable":** `combinesWith.* = false`. Tagged with metafield `node_kind: "non_combinable"`.

Both share one compiled Wasm. The Function reads `input.discountNode.metafield(namespace: "pumper", key: "node_kind").value` and filters the shop config (which carries all rules with their `combinable` flag) to return only matching rules. Strategy stays `FIRST` per node — Shopify's `combinesWith` handles cross-discount composition.

### 3.3 `matcher.rs`

Pure functions for matching cart lines against rules:

```rust
use crate::config::{Bundle, QuantityBreak, QbTier};

/// Matches a bundle against the cart. Returns target line IDs if all required products are present.
pub fn match_bundle(lines: &[CartLine], bundle: &Bundle) -> Option<Vec<String>> {
    let mut targets = Vec::new();
    for required in &bundle.products {
        let matching = lines.iter().filter(|line| {
            line.product_id == required.product_id
                && match (&required.variant_id, &line.variant_id) {
                    (Some(req_v), line_v) => Some(req_v) == line_v.as_ref(),
                    (None, _) => true,  // null variant = any variant accepted
                }
                && line.quantity >= required.qty
        });
        let line = matching.into_iter().next()?;
        targets.push(line.id.clone());
    }
    Some(targets)
}

/// For a single cart line, find the highest tier where line.quantity >= tier.qty.
pub fn match_qb_tier<'a>(line: &CartLine, qb: &'a QuantityBreak) -> Option<&'a QbTier> {
    if line.product_id != qb.product_id {
        return None;
    }
    qb.tiers.iter()
        .filter(|t| line.quantity >= t.qty)
        .max_by_key(|t| t.qty)
}

pub struct CartLine {
    pub id: String,
    pub product_id: String,
    pub variant_id: Option<String>,
    pub quantity: u32,
}
```

### 3.4 `discount.rs`

Pure discount math:

```rust
use crate::config::{Bundle, QbTier};

pub fn compute_bundle_value(bundle: &Bundle, line_subtotal: f64) -> DiscountValue {
    match bundle.discount_type.as_str() {
        "percentage" => DiscountValue::Percentage(bundle.discount_value),
        "flat" => DiscountValue::FixedAmount(bundle.discount_value),
        "fixed_total" => {
            let off = (line_subtotal - bundle.discount_value).max(0.0);
            DiscountValue::FixedAmount(off)
        }
        _ => DiscountValue::Percentage(0.0),  // unknown type — no discount
    }
}

pub fn compute_qb_tier_value(tier: &QbTier, line_amount_per_unit: f64) -> DiscountValue {
    match tier.discount_type.as_str() {
        "percentage" => DiscountValue::Percentage(tier.discount_value),
        "flat" => DiscountValue::FixedAmount(tier.discount_value),
        "fixed_per_unit" => {
            let per_unit_off = (line_amount_per_unit - tier.discount_value).max(0.0);
            DiscountValue::FixedAmount(per_unit_off)
        }
        _ => DiscountValue::Percentage(0.0),
    }
}

pub enum DiscountValue {
    Percentage(f64),
    FixedAmount(f64),
}
```

### 3.5 `main.rs`

Orchestration — the only file touching Shopify Function input/output types:

```rust
use shopify_function::prelude::*;
use shopify_function::Result;
mod config;
mod matcher;
mod discount;

#[shopify_function]
fn run(input: input::ResponseData) -> Result<output::FunctionRunResult> {
    let node_kind = input
        .discount_node
        .metafield
        .as_ref()
        .map(|m| m.value.as_str())
        .unwrap_or("combinable");

    let config: config::Config = match input.shop.metafield.as_ref() {
        Some(m) => serde_json::from_str(&m.value)
            .map_err(|e| shopify_function::Error::msg(format!("Failed to parse config: {e}")))?,
        None => {
            return Ok(output::FunctionRunResult {
                operations: vec![],
            });
        }
    };

    let want_combinable = node_kind == "combinable";

    // Convert input cart lines to our internal CartLine type
    let lines: Vec<matcher::CartLine> = input.cart.lines.iter().filter_map(|l| {
        let variant = l.merchandise.as_product_variant()?;
        Some(matcher::CartLine {
            id: l.id.clone(),
            product_id: variant.product.id.clone(),
            variant_id: Some(variant.id.clone()),
            quantity: l.quantity as u32,
        })
    }).collect();

    let mut operations = Vec::new();

    // Bundle matching
    for bundle in config.bundles.iter()
        .filter(|b| b.combinable == want_combinable && b.status == "active")
    {
        if let Some(target_line_ids) = matcher::match_bundle(&lines, bundle) {
            // Construct discount operation per cart.lines.discounts.generate.run schema
            // (exact shape depends on the Shopify Functions schema generated for this target)
            operations.push(build_bundle_discount(bundle, &target_line_ids, &input.cart.lines));
        }
    }

    // QB matching
    for qb in config.quantity_breaks.iter()
        .filter(|q| q.combinable == want_combinable && q.status == "active")
    {
        for line in &lines {
            if let Some(tier) = matcher::match_qb_tier(line, qb) {
                operations.push(build_qb_discount(qb, tier, &line.id, &input.cart.lines));
            }
        }
    }

    Ok(output::FunctionRunResult { operations })
}

// build_bundle_discount and build_qb_discount: construct the precise output operation
// shape required by `cart.lines.discounts.generate.run`. Concrete shape determined at
// implementation time from the generated `schema.graphql`.
```

The exact `output::FunctionRunResult` shape comes from the Shopify-generated `schema.graphql` after `shopify app generate schema`. Two-pass implementation: scaffold the Function, run schema generation, then write the output construction in `build_bundle_discount` / `build_qb_discount`.

## 4. Discount Node Lifecycle

### 4.1 `ensureDiscountNodes` helper

`apps/admin/app/lib/discount-nodes.ts`:

```ts
import { eq } from "drizzle-orm";
import type { DB } from "~/db.server";
import { schema } from "~/db.server";

type AdminGraphqlClient = {
  graphql(query: string, options?: { variables?: unknown }): Promise<Response>;
};

export async function ensureDiscountNodes(
  admin: AdminGraphqlClient,
  db: DB,
  shopId: string,
): Promise<{ combinable: string; nonCombinable: string }> {
  const row = (
    await db.select().from(schema.shops).where(eq(schema.shops.id, shopId)).limit(1)
  )[0];

  let combinableId = row?.shopifyDiscountIdCombinable ?? null;
  let nonCombinableId = row?.shopifyDiscountIdNonCombinable ?? null;

  if (combinableId && nonCombinableId) {
    return { combinable: combinableId, nonCombinable: nonCombinableId };
  }

  const functionId = await getOrFetchFunctionId(admin);

  if (!combinableId) {
    combinableId = await createDiscountNode(admin, functionId, "combinable", true);
  }
  if (!nonCombinableId) {
    nonCombinableId = await createDiscountNode(admin, functionId, "non_combinable", false);
  }

  await db
    .update(schema.shops)
    .set({
      shopifyDiscountIdCombinable: combinableId,
      shopifyDiscountIdNonCombinable: nonCombinableId,
    })
    .where(eq(schema.shops.id, shopId));

  return { combinable: combinableId, nonCombinable: nonCombinableId };
}

async function getOrFetchFunctionId(admin: AdminGraphqlClient): Promise<string> {
  const res = await admin.graphql(
    `query { shopifyFunctions(first: 25, apiType: "discount") { nodes { id title } } }`,
  );
  const data = (await res.json()) as {
    data: { shopifyFunctions: { nodes: { id: string; title: string }[] } };
  };
  const fn = data.data.shopifyFunctions.nodes.find((n) => n.title === "Bundler discount");
  if (!fn) {
    throw new Error(
      "Bundler discount Function not found. Run `shopify app deploy` first.",
    );
  }
  return fn.id;
}

async function createDiscountNode(
  admin: AdminGraphqlClient,
  functionId: string,
  kind: "combinable" | "non_combinable",
  combines: boolean,
): Promise<string> {
  const res = await admin.graphql(
    `mutation Create($d: DiscountAutomaticAppInput!) {
      discountAutomaticAppCreate(automaticAppDiscount: $d) {
        automaticAppDiscount { discountId }
        userErrors { field message }
      }
    }`,
    {
      variables: {
        d: {
          title: kind === "combinable" ? "Bundler (combinable)" : "Bundler (non-combinable)",
          functionId,
          startsAt: new Date().toISOString(),
          combinesWith: {
            productDiscounts: combines,
            orderDiscounts: combines,
            shippingDiscounts: combines,
          },
          metafields: [
            {
              namespace: "pumper",
              key: "node_kind",
              type: "single_line_text_field",
              value: kind,
            },
          ],
        },
      },
    },
  );
  const data = (await res.json()) as {
    data: {
      discountAutomaticAppCreate: {
        automaticAppDiscount: { discountId: string } | null;
        userErrors: { field: string[]; message: string }[];
      };
    };
  };
  const result = data.data.discountAutomaticAppCreate;
  if (result.userErrors.length > 0) {
    throw new Error(`discountAutomaticAppCreate failed: ${JSON.stringify(result.userErrors)}`);
  }
  if (!result.automaticAppDiscount) {
    throw new Error("discountAutomaticAppCreate returned null discount");
  }
  return result.automaticAppDiscount.discountId;
}
```

## 6. Risks & Contingencies

### Risk 1: `cart.lines.discounts.generate.run` API output shape mismatch
- **Probability:** low-medium.
- **Symptom:** `shopify app function run` fails schema validation on Function output.
- **Contingency:** Pivot to legacy `cart.discounts.run` per CLAUDE.md §6. Update target in `shopify.extension.toml` + reshape outputs in `main.rs`. Cost: ~half a day.

### Risk 2: Function instruction budget
- ~5M instructions per Function run. Linear scan over many rules + filtering + matching could approach this for large merchants.
- **Probability:** low for v1 (most merchants <50 rules).
- **Contingency:** Group B item 3 (pre-indexing) when it bites. Defer.

### Risk 3: Function deploy propagation delay
- After `shopify app deploy`, ~30s before `shopifyFunctions` query returns the new Function.
- **Probability:** low (merchants don't watch deploys).
- **Contingency:** Surface "App is updating, try again" message. Skip retry UX for v1.

### Risk 4: Two discount nodes confuse merchants in admin Discounts list
- They see two auto-discount entries with similar names.
- **Probability:** medium (UX, not bug).
- **Contingency:** Polaris banner explaining; defer to Phase 8.

### Risk 5: Test store missing multi-variant products
- Some matcher tests need a multi-variant product.
- **Mitigation:** Verify dev store inventory before smoke test; add a multi-variant product if needed.

### Risk 6: `shopify_function` Rust crate version drift
- API has changed across versions.
- **Mitigation:** Pin in `Cargo.toml`; update only intentionally.
